At the time of writing, there are tags that can be used in an HTML document like any other html tag.

Scripting Tags

<py-script>

The core tag for running python within the browser. Any valid python code here will be run. See any of the examples.

Code:

<py-script>
num = 4 + 4
sentence = f"I see {num} components on this page"
print(sentence)
</py-script>

Result:

Results from the above code will appear here when the code finishes running. num = 4 + 4 sentence = f"I see {num} components on this page" PyScript.write("py-script-result", sentence, append=False)

<py-repl>

Adds a python REPL to the page, where users can enter and run code. See the REPL and REPL2 examples.

Code:

<py-repl auto-generate="true">
first = "You can pre-enter code in a py-repl tag and it will be pre-populated."
second = "This can include multiple lines of code."
print(f"{first} {second}")
</py-repl>

Result:

first = "You can pre-enter code in a py-repl tag and it will be pre-populated." second = "This can include multiple lines of code." print(f"{first} {second}")

<py-env>

Used to specify additional packages that should be available in the virtual environment for other PyScript tags to import. Note that packages from the Python standard library, the PyScript package itself, and Pyodide are already in the environment by default and do not need to be included via Py-Env and See any of the Visualizations and Dashboards examples.

Code:

<py-env>

</py-env>

<py-script>
import rsa
(pubkey, privkey) = rsa.newkeys(512)
PyScript.write("py-env-result", f"{pubkey} {privkey}")
</py-script>

Result:

Results from the above code will appear here when the code finishes running. import rsa (pubkey, privkey) = rsa.newkeys(512) PyScript.write("py-env-result", f"{pubkey} {privkey}")
